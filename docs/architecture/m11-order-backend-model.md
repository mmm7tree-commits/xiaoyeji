# M11 订单后端收口 + 订单快照冻结说明

## 1. 订单与作品分离

Milestone 11 明确保持：

- 一个作品可以产生多个订单
- 订单有自己的状态
- 订单有自己的冻结快照
- 订单状态不能覆盖作品状态

## 2. 订单后端数据结构

当前订单主结构分成 4 层：

- 基础层：
  - `id`
  - `orderNo`
  - `clientId`
  - `workId`
- 状态层：
  - `statusCode`
  - `statusText`
  - `statusHint`
  - `paymentCapabilityReady`
- 金额层：
  - `unitPrice`
  - `quantity`
  - `amount`
- 快照层：
  - `addressSnapshot`
  - `workSnapshot`

## 3. 订单创建时冻结的字段

当前至少冻结这些字段：

- 作品标题
- 模板名称
- 模板版本号
- 页数
- 照片数
- 单价
- 数量
- 订单金额
- 地址快照
- 作品快照引用

其中：

- 地址快照解决“地址后续被编辑后，旧订单仍保持原下单地址”
- 作品快照解决“作品后续变化后，旧订单仍保持原下单信息”

## 4. 当前订单状态

Milestone 11 当前正式实现：

- `待支付`
- `已关闭`

当前仍然只是未来状态占位：

- `待发货`

这意味着：

- 当前创建订单后会进入 `待支付`
- 当前可以手动关闭订单进入 `已关闭`
- 当前不会伪装成已经完成支付

## 5. 用户端与后端的关系

M11 起，用户端以下链路优先读后端：

- 确认订单页创建订单
- 我的订单列表
- 订单详情

当前仍保留一层本地缓存兜底，避免接口异常时页面完全不可用。

## 6. 当前边界

M11 只做到：

- 订单创建接口后端化
- 订单列表接口后端化
- 订单详情接口后端化
- 关闭订单接口后端化

当前仍未做：

- 真支付
- 支付回调
- PDF
- 后台订单联动
- 物流
- 退款售后
