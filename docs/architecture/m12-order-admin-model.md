# M12 后台订单管理骨架 + 状态对齐说明

## 1. 这轮目标

Milestone 12 只做：

- 后台订单列表页
- 后台订单详情页骨架
- 后台读取后端订单数据
- 用户端与后台订单状态字段对齐
- 明确统一的 `statusCode` 枚举

## 2. 用户端与后台统一状态枚举

当前统一使用：

- `pending_payment`
- `pending_shipment`
- `closed`

其中：

- `pending_payment`：正式实现
- `closed`：正式实现
- `pending_shipment`：未来状态占位

用户端和后台可以有不同文案，但底层状态值必须保持一致。

## 3. 后台订单列表字段

后台订单列表当前展示：

- 订单号
- 作品标题
- 模板名称
- 模板版本号
- 收货人
- 收货手机号
- 状态
- 单价
- 数量
- 金额
- 图片是否具备履约条件
- 更新时间

## 4. 后台订单详情字段

后台订单详情当前展示：

- 订单基础信息
- 地址快照
- 作品快照摘要
- 模板版本号
- 图片资产履约准备情况
- 当前状态说明

## 5. 当前正式实现与未来占位

当前正式实现：

- 后端订单列表读取
- 后端订单详情读取
- 后台订单骨架页
- 用户端和后台状态值对齐

当前仍然只是未来占位：

- `pending_shipment` 状态流转
- 真支付
- 支付回调
- 物流
- PDF
- 退款售后

## 6. 本地订单缓存当前职责

本地订单缓存当前只保留：

- 订单列表接口失败时兜底
- 订单详情接口失败时兜底
- 兼容旧页面还没完全切走后端时的短期过渡

它不再是订单主存储。
