# Milestone 6：我的页与订单列表收口

## 我的页未登录态与已登录态展示逻辑

### 未登录态

当前“我的”页在未登录时只展示：

- 欢迎头部
- 微信授权按钮
- “登录后可以做什么”说明卡
- 联系客服 / 运营后台 / 返回作品集入口

未登录时，不直接展示个人订单列表，也不把“我的作品 / 我的订单 / 我的地址”作为可直接查看的个人数据页。

如果用户在“我的”页点击这些个人入口，会先提示登录。

### 已登录态

当前“我的”页在已登录时展示：

- 用户头像与昵称
- 订单状态概览卡
- 我的作品 / 我的订单 / 我的地址 入口
- 联系客服 / 运营后台 / 返回作品集入口

这里的定位是“用户中心入口页”，不再把完整订单列表直接堆在“我的”首页里。

## 我的订单列表页字段结构

当前订单列表页展示的数据来自订单模型，列表项会格式化成以下展示字段：

```js
{
  id,
  orderNo,
  workTitle,
  statusCode,
  statusText,
  statusClass,
  createdText,
  amountText,
  quantityText,
  previewMetaText,
  nextStepText
}
```

其中：

- `previewMetaText`：来自作品快照，展示模板名、预览页数、照片数
- `nextStepText`：当前状态说明，用于给后续支付接入预留更清晰的位置

## 订单列表与订单详情的跳转关系

当前关系如下：

- 我的页
  - 点击“我的订单”进入订单列表页
  - 点击订单状态概览卡，也会带筛选条件进入订单列表页

- 订单列表页
  - 点击“查看订单详情”进入订单详情页

- 订单详情页
  - 优先 `navigateBack`
  - 如果不是从列表栈进入，则回退到订单列表页

## 空态与返回路径规则

### 我的页

- 未登录：展示授权引导
- 已登录但没有默认地址：入口文案显示“还没有默认地址”
- 已登录但没有订单：在我的订单列表页展示空态，而不是在我的首页堆空列表

### 我的订单列表页

- 未登录：显示登录引导
- 已登录但没有订单：显示“还没有订单”空态
- 已登录且筛选后无数据：显示对应状态的空态文案

### 订单详情页

- 支持回看作品预览
- 支持回到订单列表
- 若原作品已经不在作品列表，仍保留订单快照，不强依赖作品本体

## 当前哪些订单状态仍然只是占位

Milestone 6 仍然只有这 3 个最小状态：

- `待支付`
- `待发货`
- `已关闭`

其中：

- `待支付`：只是订单创建后的占位状态，不代表已接通支付
- `待发货`：只是未来支付成功后的预留状态，不代表已接入发货履约
- `已关闭`：只是取消 / 超时关闭占位，不代表已接入真实退款

因此，当前仍然**不属于已完成支付链路**的能力包括：

- 真实支付
- 支付回调
- 物流履约
- 退款售后
- 后台订单联动

## 当前已可直接迁移到 apps/miniapp 的模型

当前这些模型已经可以从 legacy 微信小程序直接迁移到 `apps/miniapp`：

- 用户中心入口关系
- 订单状态筛选结构
- 订单列表项格式化结构
- 订单详情页使用的订单快照结构
- 地址默认状态展示规则

后续迁移时，主要替换的是页面实现和平台能力，不需要重新设计这些状态与字段。
