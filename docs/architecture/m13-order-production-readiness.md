# M13 订单可生产性校验 + 履约准备信息说明

## 1. 这轮目标

Milestone 13 只做：

- 后台订单详情里明确展示订单可生产性 / 可履约性判断
- 明确订单可生产性规则
- 文档化哪些条件满足后才具备后续履约准备
- 用户端订单详情页状态提示与说明文案收口
- 为后续支付 / PDF / 履约按钮预留位置

## 2. 订单可生产性判断规则

当前订单只有在以下 4 个条件都满足时，才算具备后续生产条件：

1. 图片资产齐全
2. 作品快照完整
3. 模板版本完整
4. 地址快照完整

### 2.1 图片资产齐全

要求：

- 作品图片已经具备服务端资源引用
- 后续可以继续给 PDF / 履约使用

### 2.2 作品快照完整

要求：

- 作品标题完整
- 模板标识完整
- 已生成页数大于 0
- 照片数大于 0

### 2.3 模板版本完整

要求：

- `templateVersionId` 存在，或
- `templateVersionNo` 存在

### 2.4 地址快照完整

要求：

- 收货人
- 手机号
- 地区
- 详细地址

## 3. 哪些缺失会阻止后续 PDF / 履约

如果以下任意一项缺失，当前都不应该继续进入后续 PDF 或履约动作：

- 图片资产还没有持久化到服务端
- 作品快照不完整
- 模板版本不完整
- 地址快照不完整

## 4. 后台订单详情新增字段

M13 新增并重点展示：

- `productionReadiness.ready`
- `productionReadiness.label`
- `productionReadiness.description`
- `productionReadiness.blockers`
- `productionReadiness.checks`

后台仍然保留：

- `assetReadiness`

因为“图片是否已具备履约条件”仍然是最直观、最关键的一层。

## 5. 用户端订单详情说明收口

用户端当前不直接做履约动作，只做：

- 当前状态说明
- 当前订单是否具备后续生产条件
- 每一项检查结果
- 后续支付 / PDF / 履约按钮占位

## 6. 当前正式实现与未来占位

当前正式实现：

- 后端返回订单可生产性判断
- 后台详情展示生产条件
- 用户端详情展示生产条件

当前仍然只是未来占位：

- 真支付
- 支付回调
- PDF 生成
- 发货
- 物流
- 退款售后
