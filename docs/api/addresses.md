# 地址接口说明（Milestone 10）

当前地址接口已经开始正式后端化，用户端确认订单页会优先读取这些接口返回的数据。

## 设备标识

所有地址接口当前都需要请求头：

- `X-Xiaoyeji-Client-Id`

当前它用于区分不同设备上的地址数据，仍然是开发阶段的设备级隔离方案。

## 1. 地址列表

### `GET /api/addresses`

返回字段：

- `id`
- `name`
- `phone`
- `region`
- `regionParts`
- `detail`
- `isDefault`
- `updatedAt`

说明：

- 有默认地址时，确认订单页会自动带出
- 没有默认地址时，确认订单页会提示先新增或设为默认

## 2. 新增地址

### `POST /api/addresses`

请求体：

```json
{
  "name": "王妈妈",
  "phone": "13800138000",
  "region": "上海市浦东新区",
  "regionParts": ["上海市", "上海市", "浦东新区"],
  "detail": "向日葵路 88 号 3 单元 501",
  "isDefault": true
}
```

返回字段：

- 地址完整结构

## 3. 编辑地址

### `PUT /api/addresses/:id`

请求体与新增地址保持一致。

返回字段：

- 地址完整结构

## 4. 设为默认地址

### `POST /api/addresses/:id/default`

返回字段：

- 地址完整结构

## 5. 删除地址

### `DELETE /api/addresses/:id`

返回字段：

- `id`
- `deleted`

## 默认地址规则

- 若存在默认地址，确认订单页自动带出这条地址
- 若删除的是默认地址，且仍有其他地址，则自动补一条默认地址
- 若删除的是最后一条地址，则当前无默认地址

## 当前边界

Milestone 10 只做到：

- 地址列表
- 新增地址
- 编辑地址
- 删除地址
- 默认地址逻辑

当前还没有做：

- 地址与订单的正式后端联动
- 多收件人复杂校验
- 地址导入后的云端去重
