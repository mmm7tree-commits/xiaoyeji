# 小页记页面深度测试记录

测试日期：2026-04-07

## 测试范围

- 首页：授权、模板选择、照片上传、生成作品。
- 作品集：作品预览、下单、删除。
- 效果预览：模板参考、自动排版预览、下单入口。
- 确认订单：地址校验、数量调整、支付模拟。
- 地址管理：微信导入、手动保存、删除。
- 我的：订单状态、订单筛选、继续支付、地址与客服入口。
- 后台：数据概括、模板管理、作品集管理、订单管理、PDF 下载占位。

## 发现并已修复的问题

- P1：作品集保存的是 `tempFilePath`，照片可能在小程序回收临时文件后失效。已改为上传后调用 `wx.saveFile` 保存本地持久文件，失败时保留临时路径以不阻断当前预览。
- P1：下单后选择“稍后支付”没有继续支付入口，购买链路中断。已在“我的”订单卡片中加入待支付订单的“继续支付”入口。
- P1：订单提交没有防重复点击，可能重复创建订单。已加入 `isSubmitting`、按钮 loading 和 disabled。
- P2：上传照片时没有保存中状态，用户可能重复触发上传。已加入 `isSavingPhotos`、保存中 loading 和 60 张上限提示。
- P2：生成作品时没有处理中状态。已加入 `isGenerating` 和按钮 disabled。
- P2：地址手机号未校验，删除地址无确认。已加入手机号校验和删除确认。
- P2：图片列表未使用懒加载。已为模板、照片、作品封面、预览图、头像加入 `lazy-load`。
- P2：预览页多处固定宽高容易在窄屏溢出。已调整预览内容宽度为百分比或 `calc`，减少横向溢出。
- P2：订单状态卡不可交互。已改为“全部/待支付/制作中/已完成”筛选。
- P2：后台作品集和订单列表没有按模板维度筛选。已加入模板筛选，并让批量 PDF 下载基于当前筛选结果。
- P3：模板长标题、来源名称、作品标题可能撑破卡片。已加入单行省略或两行截断。

## 当前未完成或依赖外部能力

- 微信支付仍是模拟流程，正式环境需要后端预支付接口和 `wx.requestPayment`。
- 物流状态仍是本地模拟，正式环境需要后端订单与物流接口同步。
- 高清打印 PDF 仍是占位，正式环境建议服务端基于原图和 PSD/版式规则生成。
- 微信开发者工具 CLI 预览未完成，原因是本机 IDE 服务端口超时；需在工具“设置 -> 安全设置”开启服务端口后复测。
